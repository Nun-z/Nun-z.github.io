Source Code and FAQ

Where can I find the source code for these applications?

Each application is publicly hosted on GitHub.  Here's a link to the RA Emulator code: https://github.com/Nun-z/RetroArch/tree/stable.  Here's a link to the RA Emulator (Unstable) code: https://github.com/Nun-z/RetroArch/tree/master.  Here's a link to the PSP Emulator code: https://github.com/Nun-z/ppsspp/tree/master.  Here's a link to the FTP Server code: https://github.com/Nun-z/DURANGO-FTP.  Here's a link to the PSX Emulator source code: https://github.com/Nun-z/duckstation.

Where do you get these packages from?

I create the packages myself from the source code linked above.  I have an entire infrastructure dedicated to making these packages and automatically uploading them to the Microsoft Store, which you can see here: https://github.com/Nun-z/AzureXboxScripts and here: https://github.com/Nun-z/NodeAzureUpptime.  I hope that it is obvious that the assertion that I take my packages from other, already existing packages makes no sense in light of the extensive work I have done to automate this process.

How are you doing this?

Here's a guide to doing this manually: https://github.com/Nun-z/XboxRetailModeAppGuide Note that the process is difficult at first and can be a struggle to complete.  In the Pricing and Availability settings, set visibility to "public audience" and set discoverability to "direct link only."

How does this compare to other options?

You don't need to put your email into a whitelist in order to install them.  Unlike other public versions, these apps can be updated, so there's no need to install a new app when a new version is released, as long as the apps haven't been taken down, which does happens periodically.  You can use any device with a web browser to install these apps to your Xbox (rather than going to the website on your Xbox's browser).  As for performance, here's a comparison video between the gamr13 retail version and my RetroRev Unstable retail version: https://youtu.be/SjBYJme3RQs.  I am always researching ways to improve performance and the user experience.

Does the auto-update feature actually work?

Yes, it does.  I've seen a lot of confusion about this, so allow me to clarify how this works.  From the edit pipeline page in Azure DevOps, select the "trigger" menu option as demonstrated here: https://i.imgur.com/AoGGkUo.png.  Then, click the "Add" button next to "Scheduled" and select a time and date as demonstrated here: https://i.imgur.com/rb07PVy.png.  The add button adds the schedule to the pipeline.  You then press the "Save" button.  The save button saves the schedule.  I hope that this helps you understand.
